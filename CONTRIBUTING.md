# Contributing

Thanks for contributing to Chess Move Notes.

## Before you start

- Please search existing issues before opening a new one.
- Small, focused PRs are easiest to review.

## Development setup

```bash
npm ci
npm run check
npm run test:run
npm run build
```

## Code style

- Formatting is handled by Prettier and linting by ESLint.
- Run `npm run check` before submitting a PR.

## Pull requests

- Describe the problem and the fix.
- Include tests when possible.
- Keep changes scoped to a single concern.

## Reporting issues

- Bugs and feature requests: open a GitHub issue.
- Security issues: see SECURITY.md.
